# LogBlade 0.1.0 Beta 2

This beta expands LogBlade's parser, search, export, and window-management workflows while preserving the portable Windows x64 distribution.

## Highlights

- Save the processed log or final search results with `Ctrl+S`; search exports include headers and captured columns.
- Add fixed parser Filter stages that feed cascading searches without recalculating completed upper levels.
- Use one output-template syntax across Regex, JSON, and Regex Replace: `$0`, `$1`, `${name}`, transformations, `$$`, and escaped control or Unicode characters.
- Run supported patterns with the culture-invariant .NET `NonBacktracking` Regex engine and open the new in-app Regex reference from the `.*` button.
- Copy search results with their headers using `Ctrl+Shift+C` and benefit from faster copying and parser-filter navigation.
- Restore selected-row focus after returning to the app, use native `Esc`/`Enter`/`Tab` behavior in auxiliary windows, and close all auxiliary windows with the main window.
- Resize visible result panes proportionally, preserve one-line pane minimums, and clip undersized layouts from the bottom without changing their configured proportions.

## Compatibility note

Parser output templates created with Beta 1 syntax must be updated: use `$0` or `$1` instead of `{0}` or `{1}`, and `${name}` instead of `{name}`. Use `$$` for a literal dollar sign.

## Install

Download `LogBlade-0.1.0-beta.2-win-x64.zip`, verify its hash against `SHA256SUMS.txt`, extract it, and run `LogBlade.exe`.

This beta is portable and supports Windows 10 and Windows 11 on x64 processors. It does not include an installer, automatic updates, or file associations. The executable is not digitally signed, so Windows SmartScreen may show an unrecognized-app warning.

# LogBlade 0.1.0 Beta 2 — Português

Esta beta amplia os fluxos de parser, busca, exportação e gerenciamento de janelas do LogBlade, preservando a distribuição portátil para Windows x64.

## Destaques

- Salve o log processado ou os resultados finais da busca com `Ctrl+S`; exportações de busca incluem cabeçalhos e colunas capturadas.
- Use estágios fixos de Filter no parser, alimentando buscas em cascata sem recalcular níveis superiores já concluídos.
- Use uma única sintaxe de templates em Regex, JSON e Regex Replace: `$0`, `$1`, `${name}`, transformações, `$$` e escapes de controle ou Unicode.
- Execute patterns compatíveis com o engine Regex `NonBacktracking` e `CultureInvariant` do .NET e abra a nova referência Regex pelo botão `.*`.
- Copie resultados de busca com cabeçalhos usando `Ctrl+Shift+C`, com melhorias de desempenho na cópia e navegação de filters do parser.
- Recupere o foco da linha selecionada ao voltar ao aplicativo, use o comportamento nativo de `Esc`/`Enter`/`Tab` nas janelas auxiliares e feche todas elas junto com a janela principal.
- Redimensione panes visíveis proporcionalmente, preserve o mínimo de uma linha e recorte layouts pequenos pela borda inferior sem alterar as proporções configuradas.

## Compatibilidade

Templates de saída criados com a sintaxe da Beta 1 precisam ser atualizados: use `$0` ou `$1` no lugar de `{0}` ou `{1}`, e `${name}` no lugar de `{name}`. Use `$$` para produzir um cifrão literal.

## Instalação

Baixe `LogBlade-0.1.0-beta.2-win-x64.zip`, confira o hash usando `SHA256SUMS.txt`, extraia o conteúdo e execute `LogBlade.exe`.

Esta beta é portátil e suporta Windows 10 e Windows 11 em processadores x64. Ela não inclui instalador, atualização automática ou associação de arquivos. O executável não possui assinatura digital, portanto o Windows SmartScreen pode exibir um aviso de aplicativo não reconhecido.
